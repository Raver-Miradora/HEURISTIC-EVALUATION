Evaluation Results Overview

This heuristic evaluation was conducted by a team of IT undergraduate evaluators to assess the usability of the Computer Laboratory Management System. Applying Jakob Nielsen's 10 Usability Heuristics, we systematically identified interface challenges and improvement opportunities across core functionalities including equipment borrowing, inventory tracking, and user management. The assessment focused on optimizing efficiency and reliability for primary users - lab administrators handling daily operations and staff members borrowing resources. Our findings aim to enhance both the technical performance and user experience of the system, ensuring it effectively supports the institution's resource management needs while maintaining intuitive operation for all user levels.

1 Visibility of System Status
Issues:

After login, the system displays a loading animation, but it plays too quickly (less than 1 second) and instantly transitions to the dashboard. Users may miss the feedback, creating uncertainty about whether their login was successful. Also, no persistent indicator like success message confirms the action.

Recommendations:

Ensure the login loading animation lasts at least 2â€“3 seconds and pair it with a clear success message before redirecting. Persistent visual feedback, like a welcome banner, should reinforce the action. Avoid instant transitions to prevent user uncertainty and enhance trust in the system.

2 Match Between System and the Real World
Issues:

The dashboard displays "Welcome, Administrator Admin!". A repetitive greeting that mismatches real-world. This unnatural phrasing fails to reflect real-world communication norms, potentially confusing users. Additionally, the redundant use of "Administrator Admin" appears unprofessional and may undermine user trust in the system's attention to detail.

Recommendations:

To improve usability, the greeting should adopt natural language, such as "Welcome, [First Name]!" or "Hello, [Role/Name]!". This change would align with real-world conventions, enhance clarity, and personalize the user experience. Ensuring distinct, role-specific greetings would also maintain professionalism while making the interface more intuitive.

3 User Control and Freedom
Issues:

The system confirms actions such as deletions but lacks undo/redo functionality. Users cannot recover accidentally deleted items without tedious steps. This violates the heuristics #3 since users must "go backward" manually, increasing frustration and effort for reversible actions.

Recommendations:

To improve user control and freedom, the system should implement an "Undo" option and introduce a recycle bin/archive system for recoverable items. Additionally, keyboard shortcuts (Ctrl+Z/Y) should be supported for quick reversals.

4 Consistency and Standards
Issues:

The dashboard uses the same icon for both Categories and Pending Borrowed sections, creating confusion since icons should represent distinct functions. Additionally, deletion pop-ups lack standard red buttons for destructive actions, violating consistency.

Recommendations:

To improve consistency, use distinct icons and standard red buttons for deletions. Follow platform conventions for colors and layouts. Add tooltips to clarify icons.

5 Error Prevention
Issues:

The system displays effective error messages but relies too heavily on corrections after mistakes occur. It lacks proactive safeguards like confirmation prompts for high-risk transactions, and input constraints for fields like dates. This reactive approach forces users to fix preventable errors, increasing frustration.

Recommendations:

Implement auto-formatting for account/phone numbers, add confirmation dialogs for large transfers, and use input masks for fields like dates. Disable submission until all fields are valid and highlight discrepancies in real-time.

6 Recognition rather than Recall
Issues:

The system uses dropdowns for selections but lacks breadcrumbs, forcing users to recall their navigation path instead of recognizing it visually. This violates Heuristic #6 by relying on memory over visible cues, increasing cognitive load and potential errors during multi-step processes. Users must remember their journey rather than seeing it displayed.

Recommendations:

The system should add breadcrumbs to show navigation paths and maintain recently selected dropdown items at the top. Include clear labels, autocomplete suggestions, and visual history cues.

7 Flexibility and Efficiency of use
Issues:

The system lacks keyboard shortcuts for faster navigation and allows limited personalization (logo/name changes) but doesn't let users customize functionalities like dashboard layouts or workflow preferences. This reduces efficiency for power users who rely on accelerators or tailored interfaces.

Recommendations:

Add keyboard shortcuts and customizable workflows to boost efficiency. Enable personalized dashboards and saved templates while maintaining simplicity. These accelerators will serve both novice and expert users without complicating the interface

8 Aesthetic and Minimalist Design
Issues:

The system already maintains a clean interface by prioritizing essential content and removing unnecessary elements. However, excessive minimalism risks hiding useful featuresâ€”users may overlook advanced functions if the design becomes overly sparse or lacks visual hierarchy.

Recommendations:

Balance minimalism with discoverability: use subtle visual cues (e.g., icons, micro-interactions) to guide users to advanced features without clutter. Maintain whitespace but ensure critical actions remain immediately apparent through strategic contrast or progressive disclosure techniques.

9 Help Users Recognize, Diagnose, and Recover from Errors
Issues:

The system displays errors using only red text without clear solutions, leaving users confused about how to fix problems. Generic messages like "Invalid input" fail to explain causes or corrective actions, increasing frustration during task completion.

Recommendations:

Replace generic errors with actionable messages (e.g., "Password must contain 8 characters"). Add inline corrections (e.g., auto-formatting) and a "Help" button linking to specific resolution steps when errors occur.

10 Help and Documentation
Issues:

The system lacks essential help documentation and in-app guidance, forcing users to rely on trial-and-error or external support. This violates Heuristic #10, creating frustration when users encounter unfamiliar features or errors without accessible reference material.

Recommendations:

Implement contextual tooltips, a searchable knowledge base, and step-by-step guides within the interface. Ensure documentation is easily accessible without leaving the system, with clear troubleshooting sections for common issues. Prioritize user-friendly, just-in-time assistance.

Conclusion of the Evaluation

The heuristic evaluation revealed several usability challenges across the Computer Management System, primarily concerning feedback clarity, intuitive design, and user control. Key findings indicate inconsistencies in visual cues, insufficient error prevention measures, and limited user guidance, which collectively hinder the user experience.

The system would benefit from enhanced feedback mechanisms, such as clearer success indicators and actionable error messages, along with improved navigation aids like breadcrumbs and distinct icons. Implementing undo functionality, customizable workflows, and contextual help documentation would further empower users and streamline interactions.

To address these issues, we recommend prioritizing intuitive design updates, including standardized visual elements and proactive safeguards, while maintaining a balance between minimalism and functionality. These enhancements will create a more efficient, user-friendly experience for both novice and expert users.
